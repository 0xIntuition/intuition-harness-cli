# MetaStack Workspace

This directory stores repo-local MetaStack state generated by `metastack-cli`.

- `meta.json` stores repo-scoped defaults such as the Linear team and project ID.
- `backlog/` holds team task drafts and reusable backlog templates.
- `backlog/_TEMPLATE/` stores the repo-local backlog template used by `meta plan` and `meta technical`.
- `agents/briefs/` stores deterministic kickoff prompts created by `meta agents kickoff`.
- `agents/sessions/` stores repo-local agent logs such as `meta scan`; MetaListen session state is
  stored in the install-scoped MetaStack data root keyed by the canonical source `.metastack` root.
- `codebase/` holds generated repository context for future agents.
- `cron/` stores Markdown cron jobs plus repo-local scheduler runtime state.
- `workflows/` stores repo-local workflow playbooks consumed by `meta workflows`.

Typical flow:

1. Run `meta scaffold`
2. Run `meta config`
3. Run `meta scan`
4. Run `meta agents kickoff --ticket <ID>`
5. Run `meta listen`
